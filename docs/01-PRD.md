# 01. 제품 요구사항 정의 (PRD)

## 1. 문제 정의

전국 약 50만 교사는 수업 자료를 찾기 위해 평균 5개 이상의 사이트를 매번 따로 방문하고, 각각 로그인하고, 각각 검색해야 합니다. 자료 한 건을 찾는 데 평균 15~30분이 소요되며, 이는 학생 지도와 수업 연구에 써야 할 시간을 잠식합니다.

### 핵심 통증 포인트
- 5개 사이트 각각 로그인 유지·비밀번호 관리 부담
- 사이트마다 검색 인터페이스가 달라 검색 노하우가 분절됨
- 같은 단원 자료가 여러 곳에 흩어져 있어 비교·선택에 시간 소요
- 모바일에서 다중 사이트 탐색은 사실상 불가능

## 2. 솔루션

**쌤서치**는 교사가 자주 사용하는 5개 사이트의 ID/PW를 안전하게 등록해두면, 단 한 번의 검색으로 5개 사이트의 결과를 통합 리스트로 보여주는 메타 검색 서비스입니다.

### 핵심 가치 제안
> 한 번의 검색으로 수업 준비 시간을 1/6로 줄여드립니다.

## 3. 타겟 사용자

### 1차 타겟 (MVP)
- 초등 교사 (인디스쿨 14만 회원 풀)
- 자료 검색에 주 5시간 이상 사용하는 헤비 유저
- 디지털 도구에 익숙한 30~40대 교사

### 2차 타겟 (확장)
- 중·고등 교사 (티쳐빌, T셀파 중심)
- 특수교사, 비교과 교사
- 사범대 예비 교사

## 4. 페르소나

### 김선생 (33세, 초등 5학년 담임, 5년차)
- 매일 아침 30분 일찍 출근해 그날 수업자료 검색
- 인디스쿨, 아이스크림, 에듀넷 주로 이용
- 검색 결과를 PPT로 다운로드해서 본인 수업에 맞게 편집
- "좋은 자료를 누가 어디 올렸는지 한눈에 보고 싶다"

### 박교사 (42세, 중학교 사회 교사, 15년차)
- 티쳐빌·T셀파 유료 회원
- 수업 연구 시간이 부족해 검색 효율을 중시
- "여러 사이트 결과를 비교해서 가장 좋은 자료를 골라 쓰고 싶다"

## 5. 핵심 기능

### F-1. 회원가입 및 OTP 인증
- 이메일 + 비밀번호 + 핸드폰 본인인증
- SMS OTP 또는 TOTP(구글 OTP) 선택
- 디바이스 신뢰 30일 (재OTP 면제)

### F-2. 외부 계정 연결
- 5개 사이트 각각 ID/PW 등록 인터페이스
- 등록 시 OTP 1회 추가 인증
- 등록 즉시 자동 로그인 테스트 → 성공 시에만 저장
- 연결 상태 실시간 표시 (✓ 연결됨 / ⚠ 재인증 필요)

### F-3. 통합 검색
- 단일 검색창 + 필터 (학년 / 과목 / 자료유형 / 기간)
- 5개 사이트 병렬 검색 (4초 이내 응답 목표)
- 스트리밍 UI: 먼저 도착한 사이트부터 즉시 표시
- AI(Gemini)가 결과를 공통 스키마로 구조화

### F-4. 결과 표시
- 카드 형태, 사이트 식별 아이콘 표시
- 제목 / 요약 / 작성일 / 학년·과목 / 자료유형 / 좋아요 수
- "원문 보기" 클릭 시 새 탭에서 원본 사이트 이동
- 정렬: 관련도순 / 최신순 / 사이트별

### F-5. 검색 이력 및 즐겨찾기
- 최근 검색어 자동 저장 (개인용)
- 결과 카드 북마크 (URL 메타데이터만 저장)
- 검색 이력 기반 추천 (선택 기능, v2)

### F-6. 계정 관리
- 등록된 외부 계정 목록 보기·삭제
- 우리 서비스 비밀번호 변경
- 디바이스 관리 (등록된 기기 확인·해제)
- 계정 삭제 (모든 자격증명 즉시 폐기)

## 6. 비기능 요구사항

### 성능
- 통합 검색 P50 응답: 4초 이내
- 통합 검색 P95 응답: 8초 이내
- 캐시 히트 시: 0.5초 이내
- 동시 사용자 100명 대응 (MVP)

### 보안
- 외부 계정 비밀번호: AES-256-GCM 암호화 + KMS 키 분리
- 우리 서비스 비밀번호: bcrypt + salt
- 모든 통신: TLS 1.3
- OTP: 디바이스 신뢰 30일, 신규 디바이스·이상 패턴 시 재인증
- 본문 저장 금지 — 메타데이터만

### 가용성
- MVP: 99% (월 약 7시간 다운타임 허용)
- 정식 출시: 99.5%
- 사이트별 장애 격리 — 한 사이트 다운돼도 나머지 4곳은 정상 동작

### 확장성
- 사이트 어댑터 모듈화 → 신규 사이트 추가 시 1주 이내 배포
- 워커 수평 확장 (Fly.io 컨테이너 자동 스케일)

## 7. 성공 지표

### 서비스 출시 후 3개월
- 가입자 1,000명
- 외부 계정 평균 연결 수: 3개 이상
- WAU(주간 활성 사용자): 30%
- 평균 주 검색 횟수: 10회 이상

### 6개월
- 가입자 5,000명
- 유료 전환율 5% (월 4,900원 기준)
- NPS(순추천지수): 40 이상

## 8. 범위 제외 (Out of Scope - MVP)

- 자료 다운로드·저장 (메타데이터만)
- 자료 본문 미리보기 (저작권 이슈)
- AI 요약·번역 등 가공 기능 (v2 이후)
- 모바일 네이티브 앱 (반응형 웹으로 대응)
- 결제/구독 시스템 (베타 무료)
- 팀·학교 단위 계정 (v2 이후)

## 9. 가정과 제약

### 가정
- 각 사이트의 자동 로그인이 봇 차단 없이 동작
- 사이트별 검색 결과 페이지 구조가 6개월 이상 안정
- 사용자가 자기 계정의 위임 사용에 동의

### 제약
- 사이트 약관 변경 시 즉시 대응 가능한 운영 체계 필요
- 2FA 도입 사이트는 별도 흐름 필요 (현재 5개 사이트는 모두 단일 인증)
- KISA 정보보호 관리체계(ISMS) 인증은 PMF 검증 후 검토
