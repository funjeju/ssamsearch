# 11. 개발 로드맵

## 1. 마일스톤 개요

| 단계 | 기간 | 목표 |
|---|---|---|
| **Phase 0** — PoC | 1주 | 인디스쿨 1개 사이트로 핵심 흐름 검증 |
| **Phase 1** — MVP | 5주 | 5개 사이트 + 인증 + 보안 + 베타 출시 |
| **Phase 2** — 사용자 검증 | 4주 | 베타 100명, 데이터 수집, 빠른 개선 |
| **Phase 3** — 정식 출시 | 4주 | 결제, 추천 알고리즘, 모바일 최적화 |
| **Phase 4** — 확장 | 지속 | 신규 사이트 추가, B2B (학교/지자체) |

**총 MVP까지 6주, 정식 출시까지 14주.**

## 2. Phase 0 — PoC (1주)

> 목적: "기술적으로 정말 되는가?"를 1주 안에 검증.

### Day 1~2: 환경 셋업
- [ ] 모노레포 구조 생성 (apps/web, apps/worker, packages)
- [ ] Next.js 14 + Tailwind + shadcn/ui 초기화
- [ ] Firebase 프로젝트 생성, Firestore 활성화
- [ ] Playwright 설치 및 헬로월드 스크립트

### Day 3~4: 인디스쿨 어댑터
- [ ] 본인 계정으로 수동 로그인 → DOM 구조 분석
- [ ] Playwright 자동 로그인 스크립트
- [ ] 검색 페이지 호출 + Cheerio 파싱
- [ ] 결과 10개 정상 추출 확인

### Day 5: Gemini 연동
- [ ] Gemini API 키 발급
- [ ] HTML → 구조화 JSON 프롬프트 작성·튜닝
- [ ] 결과 정확도 80% 이상 확인

### Day 6~7: 최소 UI
- [ ] 검색창 + 결과 리스트 (디자인은 단순하게)
- [ ] 본인 계정 환경변수에 박아두고 e2e 동작 확인
- [ ] 응답 시간 측정 (목표: 6초 이내)

### PoC 완료 기준
- ✅ 검색어 입력 → 6초 이내 결과 표시
- ✅ AI가 구조화한 메타데이터(제목·요약·학년·과목) 80% 이상 정확
- ✅ 클릭 시 인디스쿨 원문 정상 이동
- ❌ 실패 시: 어떤 단계에서 막혔는지 파악 후 우회 방안 결정

## 3. Phase 1 — MVP (5주)

### Week 1: 인프라 + 인증

#### 백엔드
- [ ] Firebase Auth 통합 (이메일/비번 로그인)
- [ ] OTP 시스템 (SMS + TOTP)
- [ ] 디바이스 신뢰 메커니즘
- [ ] Firestore 보안 규칙 작성·테스트

#### 프론트엔드
- [ ] 디자인 시스템 셋업 (컬러, 타이포)
- [ ] 회원가입 / 로그인 / OTP 검증 페이지
- [ ] 라우팅 구조 (App Router)

#### 인프라
- [ ] Vercel 프로젝트 연결
- [ ] Upstash Redis 연결
- [ ] GCP KMS 셋업

### Week 2: 외부 계정 관리

- [ ] envelope encryption 구현·테스트 (`packages/crypto`)
- [ ] 외부 계정 등록 API (`POST /api/accounts`)
- [ ] 등록 즉시 자동 로그인 테스트 (워커 호출)
- [ ] 외부 계정 목록 UI
- [ ] 외부 계정 삭제 (OTP 검증)

### Week 3: 5개 사이트 어댑터

이전에 완성된 인디스쿨 패턴을 4개 사이트로 복제.

- [ ] 아이스크림S 어댑터 (1.5일)
- [ ] 에듀넷 어댑터 (1일, 가장 단순)
- [ ] 티쳐빌 어댑터 (1.5일)
- [ ] T셀파 어댑터 (1.5일)
- [ ] 통합 테스트 (실제 계정으로 검색)

### Week 4: 통합 검색 + 결과 UI

- [ ] BullMQ 잡 큐 구현
- [ ] Fly.io 워커 배포
- [ ] SSE 스트리밍 API
- [ ] 검색 페이지 (검색창 + 필터 + 결과 카드)
- [ ] 사이트별 진행 상태 UI
- [ ] 결과 통합·랭킹 알고리즘
- [ ] 검색 결과 캐시 (Redis)

### Week 5: 안정화 + 베타 출시

- [ ] 검색 이력·북마크 기능
- [ ] 사용자 설정 페이지 (디바이스 관리, 비번 변경)
- [ ] 에러 처리·복구 시나리오 (사이트 다운, 캡차, 비번 변경)
- [ ] Sentry 통합
- [ ] 모바일 반응형 최종 점검
- [ ] 도메인 연결 (teacherhub.kr)
- [ ] 베타 신청 페이지 + 초대 코드 시스템
- [ ] **베타 클로즈드 출시 (50명)**

### MVP 완료 기준
- ✅ 사용자가 가입 → 5개 계정 등록 → 통합 검색 → 결과 클릭 가능
- ✅ 검색 P50 응답 시간 4초 이내
- ✅ 사이트별 로그인 성공률 95% 이상
- ✅ 보안: KMS 암호화, OTP, 감사 로그 정상 동작
- ✅ Sentry에 critical 에러 0건

## 7. 리스크 관리

### 기술 리스크

| 리스크 | 가능성 | 영향 | 대응 |
|---|---|---|---|
| 사이트 봇 차단 | 중 | 높음 | residential proxy, 사용자별 IP 분산 |
| 사이트 UI 개편 | 높 | 중간 | 어댑터 모듈화, nightly E2E |
| 사이트 2FA 도입 | 낮 | 매우 높음 | TOTP 자동 입력 또는 사용자에 OTP 푸시 |
| Gemini API 변경 | 중 | 낮음 | OpenAI fallback 준비 |
| Vercel 함수 제한 | 낮 | 중간 | 워커 분리로 이미 회피 |

### 운영 리스크

| 리스크 | 대응 |
|---|---|
| 사이트 운영진 항의 | 사전 컨택, 트래픽 제어, 합법적 사용 강조 |
| 사용자 비번 유출 | KMS 암호화로 사후 피해 최소화 + 즉시 알림 |
| 사용자 폭증 | Vercel·Fly.io 자동 스케일 |
| 부정 사용 (봇 가입) | reCAPTCHA, 교사 인증 검토 |

### 비즈니스 리스크

| 리스크 | 대응 |
|---|---|
| PMF 미달성 | Phase 2에서 빠른 피보팅 (검색 → 큐레이션 등) |
| 무료 사용자만 | 프로 기능 차별화 (AI 가공, 무제한) |
| 인디스쿨이 비슷한 기능 출시 | 차별화: 5개 통합 + AI 가공 |

### Phase 2 종료 시점

- 평균 주 검색: 5회 이상

### Phase 3 종료 시점
- 가입: 5,000명
- 유료 전환: 5%
- NPS: 40 이상

## 9. 의존성 다이어그램

각 마일스톤 시작 조건:

```
Phase 0 (PoC)
   ↓ 기술 검증 성공
Phase 1 (MVP)
   ├─ Week 1 (인증) ─┐
   ├─ Week 2 (계정) ─┤
   ├─ Week 3 (어댑터)├─→ Week 4 (통합 검색) → Week 5 (베타)
   └─ Week 1~2 병행 ─┘
   ↓ 베타 출시
Phase 2 (검증)
   ↓ DAU/NPS 목표 달성
Phase 3 (정식)
   ↓ MRR 목표 달성
Phase 4 (확장)
```

## 10. 1주차 즉시 액션

PoC 시작을 위한 **이번 주 안에 할 일**:

1. 모노레포 셋업 + Next.js 14 + shadcn/ui 초기 화면
2. Firebase 프로젝트 생성, Firestore 보안 규칙 초안
3. 본인 인디스쿨 계정으로 Playwright 자동 로그인 스크립트
4. 인디스쿨 검색 페이지 DOM 구조 분석 (셀렉터 정리)
5. Gemini API 키 발급 + 첫 호출 성공

위 5개가 끝나면 PoC의 70%는 완성. 나머지는 통합과 다듬기.
